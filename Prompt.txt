PROCESO DE TRABAJO



Hazlo en fases:



FASE 1 — Análisis



* Recorre todos los repos.

* Identifica stack, dependencias, estructura y propósito.

* Crea MERGE_ANALYSIS.md.



FASE 2 — Decisión de arquitectura



* Elige el repo o stack base.

* Define la estructura final.

* Crea ARCHITECTURE.md.



FASE 3 — Migración



* Crea la nueva carpeta aio-scrap-crawl.

* Migra las mejores funciones.

* Ordena cada función en módulos.

* Elimina duplicados.

* Refactoriza lo necesario.



FASE 4 — Integración



* Unifica configuración.

* Unifica dependencias.

* Unifica scripts.

* Unifica logs.

* Unifica exports.

* Crea una API o CLI principal, según lo más conveniente.



FASE 5 — Validación



* Instala dependencias.

* Ejecuta pruebas.

* Ejecuta build si aplica.

* Corrige errores.

* Documenta lo que no se pudo corregir.



FASE 6 — Documentación final



* Crea README.md con:



  * Descripción del proyecto.

  * Instalación.

  * Uso.

  * Comandos disponibles.

  * Estructura.

  * Ejemplos.

  * Cómo agregar nuevos módulos.

* Crea MIGRATION_NOTES.md.

* Crea TODO.md.

* Crea CHANGELOG.md.
